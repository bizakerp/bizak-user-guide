# General Preferences

General preferences control how the system behaves for a company or organization.

## Typical settings

- default language
- default currency
- date format
- number format
- time zone
- numbering behavior
- tax or VAT controls
- multi-currency behavior
- rounding rules
- dashboard defaults
- notifications
- theme or display mode
- approval workflow defaults

## What these settings affect

- how dates appear
- how money amounts are shown
- what users see first after login
- whether approvals are required
- how numbers are generated
- how rounding works
- whether tax logic is active

## Example

A company in Nepal may use:

- Nepali or English labels
- NPR as the default currency
- DD-MM-YYYY date format

## Example

If the company uses Nepal time and NPR, the system should reflect that in the default settings.
If a team wants approval before posting, that should be turned on here.
