# Estimate

This page covers the estimate flow.
Use it when you want to send a quote before the order is confirmed.

## What the screen tracks

| Field | Meaning |
| --- | --- |
| `estimate_number` | Estimate number |
| `reference_from` | Source reference |
| `ref_type` | Reference type |
| `party` | Customer name |
| `address` | Billing address |
| `secondary_address` | Alternate address |
| `title` | Estimate title |
| `date` | Estimate date |
| `due_date` | Due date |
| `expected_close_date` | Expected close date |
| `memo` | Notes |
| `currency_id` | Currency |
| `exchange_rate` | Exchange rate |
| `sales_rep_id` | Sales rep |
| `forecast_type` | Forecast type |
| `partner` | Partner name |
| `department_name` | Department |
| `location` | Location |
| `class_id` | Class |
| `details` | Line items |

The estimate can also carry notes, files, tasks, events, messages, and relationships.

## What to include

- Clear item names
- Correct unit and quantity
- Rate and discount
- Tax where needed
- Project or location if required

## Example

A customer asks for a quote for 20 laptops.
The estimate lists each item, the quantity, the rate, and the expected close date.
The customer can then approve the quote or ask for changes.
