# Table

This page covers POS table setup.
Use it when the business serves customers by table.

## What the screen tracks

| Field | Meaning |
| --- | --- |
| `table_name` | Table name |
| `table_number` | Table number |
| `capacity` | Seating capacity |
| `section` | Section |
| `floor` | Floor |
| `counter_id` | Counter |
| `position_x` | X position |
| `position_y` | Y position |
| `shape` | Table shape |
| `notes` | Notes |
| `is_active` | Active flag |
| `display_order` | Sort order |

The route supports create, list, and update flows.

## Example

A restaurant creates tables for each dining area.
The POS screen can then assign orders to the right table.
