# Customization

Customization pages will document Bizak-specific configuration, labels, and business behavior changes that are safe to expose to administrators.

## Possible topics

- screen labels
- module defaults
- form behavior
- company-specific options

## What to keep stable

Customization should not break business flow.
It should keep the same core save behavior, account impact, and record structure.

## Example

If a company wants a custom label for a field, the guide should explain the label change without changing the underlying posting logic.
